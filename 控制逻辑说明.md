# 控制逻辑说明

本文档整理当前项目中电机控制主逻辑的实现思路，内容基于 [LQ_K60DN_母板测试/src/User/main.c](LQ_K60DN_母板测试/src/User/main.c) 的当前版本。

## 1. 控制目标

当前控制系统的目标是：
- 接收 UART 指令设置目标速度
- 通过编码器反馈计算当前速度
- 根据目标速度选择合适的控制方式输出 PWM
- 驱动电机转动，并在 LCD 上显示关键诊断值

## 2. 关键变量

- g_target_speed：目标速度
- g_motor_pwm：当前输出到电机的 PWM 值
- current_speed：由编码器反馈换算得到的当前速度
- encoder_count：每个控制周期内读取到的编码器脉冲数
- motor_pid：PID 控制器实例
- g_pid_enabled：是否启用 PID 控制

## 3. 控制流程

主循环流程如下：

1. 检查是否收到停止命令
   - 如果收到 'S'，立即清零目标速度、PWM 和 PID 状态
   - 直接停机并退出本轮控制

2. 读取编码器反馈
   - 每 50ms 读取一次编码器计数
   - 将其作为当前速度反馈值

3. 根据目标速度选择控制分支
   - 目标速度为 0：直接清零 PWM
   - 目标速度绝对值小于 180：走低速/中速手动 PWM 曲线
   - 目标速度绝对值大于等于 180：走高速 PID 分支

4. 输出 PWM 到驱动电路
   - 将计算出的 PWM 值送入电机驱动函数
   - 通过 FTM 输出到电机控制通道

5. 每 1 秒刷新一次 LCD 诊断显示
   - 显示目标值、当前速度、PWM、误差

## 4. 低速/中速控制分支

当目标速度绝对值小于 180 时，当前逻辑不直接使用完整 PID，而是使用一套“手动 PWM 曲线”来产生输出。

### 设计思路
- 低速时先给一个较大的起步 PWM，避免目标值有命令但电机完全不转
- 通过目标值大小决定期望 PWM
- 使用 Ramp_PWM 函数让 PWM 变化平滑，不会一步跳太多

### 当前分段规则
- 目标绝对值 <= 60：
  - 期望 PWM 约为 70 + 目标 * 1.0
- 目标绝对值 <= 140：
  - 期望 PWM 约为 130 + (目标 - 60) * 0.4
- 目标绝对值 > 140：
  - 期望 PWM 约为 148 + (目标 - 140) * 0.2

### 作用
- 解决低速起步时“有命令但不转”或者 PWM 过小的问题
- 提供更平滑的中段控制，避免直接跳到较大的输出值

## 5. 高速控制分支

当目标速度绝对值达到 180 以上时，系统切换到 PID 控制。

### 当前 PID 结构
- kp = 1.2
- ki = 0.02
- kd = 0.1

### PID 计算内容
- error = target - feedback
- 积分项带抗饱和处理
- 微分项用于减小突变
- 输出会被限制在 [-300, 300]

### 输出限制
- 低速启动时，对小误差做软限制，避免一下子冲到过大输出
- 高速分支中，输出会再被限制为较温和的范围，当前逻辑中主要限制在 [-240, 240]

## 6. PWM 输出方式

驱动函数 Drive_Set_Motor_PWM 会把 PWM 值按正反转方向分配到四个通道：
- 正向：两个通道输出占空比，两个通道保持满占空比
- 反向：相反分配
- 0：四个通道都置为零输出

## 7. LCD 诊断显示

当前每 1 秒刷新一次 LCD 诊断信息，显示内容为：
- TAR：目标速度
- CUR：当前速度
- PWM：当前 PWM
- ERR：当前误差

显示位置目前设在第 3 行和第 5 行，避免与上方已有内容重叠。

## 8. 当前控制逻辑的特点

- 采用“低速手动 PWM + 高速 PID”的混合控制策略
- 目标比 180 小的时候更强调平滑起步
- 目标比 180 大的时候切换到更稳的闭环控制
- 通过 PWM 斜坡函数减少突变

## 9. 后续调试建议

如果后续还要继续调节，可以优先关注以下几个点：

1. 低速起步阈值
   - 当前是 180
   - 如果电机在低速区仍然不够有力，可适当下调或提高起步 PWM

2. 低速曲线斜率
   - 可以适当增大起步 PWM，或者调深一点中段斜率

3. PID 参数
   - 如果高速区响应过慢或过冲，可以适当调整 kp / ki / kd

4. 输出限幅
   - 如果 250 以上仍然看起来受限，可适当提高高速分支的上限
